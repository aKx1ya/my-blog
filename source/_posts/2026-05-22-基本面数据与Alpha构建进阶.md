<h2>一、基本面数据 Fundamental Data</h2>

<p>帮助理解公司的实际经营情况，预测未来增长潜力的基础材料，是公司估值的关键要素。</p>

<p>基本面数据主要来源于：<strong>资产负债表、利润表、现金流量表</strong>。</p>

<p>特点是更新频率低。与价格/成交量不同，基本面数据是非连续更新。</p>

<hr>

<h2>二、Fundamental Data 与 PV Data 结合</h2>

<p>Fundamental data 反映公司实际的盈利能力，而 PV data（如市值）反映市场对公司价值的实时评估。二者结合可以比较<strong>"公司实际业绩"和"市场未来预期"</strong>之间的差异，从而发现定价或投资的机会。</p>

<p>例如：</p>
<ul>
  <li><strong>差值</strong>：预测与实际的差异</li>
  <li><strong>比值</strong>：营业收入/市值——每单位市值产生的收入，可用于不同规模公司间比较</li>
  <li><strong>相对变化率</strong>：(y - x) / x</li>
</ul>

<p>⚠️ <strong>需要注意量纲问题</strong>：差值不能直接横向比较，因为差值大小与公司规模有关。</p>

<hr>

<h2>三、估值指标</h2>

<h3>① EBITDA（息税折旧摊销前利润）</h3>

<p>Earnings Before Interest, Taxes, Depreciation, and Amortization</p>

<p>基本面数据，每季度/年更新。消除了资本结构和税率差异，便于跨公司比较。</p>

<h3>② EV（Enterprise Value 企业价值）</h3>

<p><strong>EV = 市值 + 总债务 + 少数股东权益 + 优先股 − 现金及现金等价物</strong></p>

<p>PV 数据，随股价更新。比市值更全面，考虑了债务和现金的影响。</p>

<h3>二者结合</h3>

<p>用除法 <strong>EV/EBITDA</strong>（盈利收益率），表示用 EBITDA 来偿还 EV 需要多少年，数值越低，估值越有吸引力。</p>

<h2>四、TS 与 CS 算子</h2>

<h3>TS Operators（时间序列）</h3>

<p>针对每只股票独立计算。当公司之间规模、量纲差异较大，横截面比较不合理时使用 TS。动量、反转策略基于自身历史变化驱动，也用 TS。</p>

<ul>
  <li><code>ts_rank(x, d)</code>：将股票 x 在过去 d 天的值排序，输出 0 ~ 1 之间的标准化排名</li>
  <li><code>ts_zscore(x, d)</code>：计算当前值与过去 d 天均值的偏离程度（以标准差为单位）</li>
  <li><code>ts_mean(x, d)</code>：过去 d 天的平均值</li>
</ul>
<!-- endtab -->

<!-- tab CS 横截面算子 -->
<h3>CS Operators（横截面）</h3>

<p>针对同一时刻所有股票计算。数据已经经过标准化处理，需要直接选出最优股票时用 CS。</p>

<ul>
  <li><code>rank(x)</code>：所有股票按 x 值排名，输出 0 ~ 1 之间的值</li>
  <li><code>zscore(x)</code>：所有股票的 x 值标准化（减均值除标准差）</li>
  <li><code>winsorize(x, std=4)</code>：将极端值截断在 ±4 倍标准差范围内</li>
</ul>
<!-- endtab -->
{% endtabs %}

{% hideToggle 💡 实战案例：一个 PASS 的 Alpha（点击展开） %}
<h3>案例对比</h3>

<p><strong>失败版本：</strong><br>
<code>ts_rank(operating_income/cap, 250)</code><br>
→ Fitness of 0.96，低于 cutoff 1.0 ❌</p>

<p><strong>PASS 版本：</strong><br>
<code>ts_rank(zscore(operating_income / cap), 250)</code><br>
→ PASS！✅</p>

<p>其中 decay = 2，truncation = 0.05</p>

<p>💡 <strong>关键启示</strong>：先对原始因子做横截面标准化（zscore），再做时间序列排名，消除了量纲差异，Alpha 质量显著提升。</p>
{% endhideToggle %}

<h2>五、子宇宙测试 Subuniverse Test</h2>

<p>这里的 Universe 就是股票池，按照交易量有 <strong>TOP3000、TOP1000、TOP500</strong> 等。数字越小流动性越高、市值越大。</p>

<p><strong>Subuniverse Test</strong> 就是把同一个 Alpha 放在不同的 Universe 上回测。如果 Alpha 在流动性最好的 TOP200 中表现依然良好，说明这个 Alpha 很稳健。</p>

<p>如果 Subuniverse Test 失败，可以通过更加集中持仓于高流动性、大盘股来提升 Subuniverse Sharpe。</p>

{% hideToggle 🛠 提升 Subuniverse 的四种方法（点击展开） %}
<ol>
  <li><strong>加入市值过滤</strong>：只保留市值较大的股票，例如 <code>cap > 1e10</code></li>
  <li><strong>加入流动性过滤</strong>：只保留日均成交额高的股票</li>
  <li><strong>使用更稳健的因子组合</strong></li>
  <li><strong>降低 Turnover</strong>：对信号做平滑处理，减少高流动性股票的过度交易</li>
</ol>
{% endhideToggle %}
