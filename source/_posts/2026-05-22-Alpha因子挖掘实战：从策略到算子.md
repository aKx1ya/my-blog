<h2>三、量化策略两大流派</h2>

{% tabs 动量 vs 反转 %}
<!-- tab 🚀 动量 Momentum -->
<h3>Momentum Strategy</h3>
<p>理解为<strong>"惯性"</strong>——Following price trend，顺势而为。价格跟随 trend 方向运行。上升趋势建立多头头寸，下降趋势做空。</p>

{% hideToggle 📈 动量效应的时间窗口特性（点击展开） %}
<p>动量并非在所有时间段都有效：</p>
<ul>
  <li><strong>短期（&lt; 1 个月）</strong>：市场情绪导致容易出现 Reversion Effect</li>
  <li><strong>中期（1 季度 ~ 1 年）</strong>：最显著！信息扩散 + 羊群效应，趋势延续</li>
  <li><strong>长期（1 年以上）</strong>：动量衰弱，基本面逐渐完全定价</li>
</ul>

<p><strong>一年期收益率动量公式：</strong><code>ts_delta(close, 250) / ts_delay(close, 250)</code></p>
<p>⚠️ 问题：短期回调噪音会干扰中期动量信号。</p>

<p><strong>优化方案 1：</strong><code>ts_delay(ts_delta(close,250)/ts_delay(close,250), 10)</code><br>
通过 delay 10 天，故意忽略最近 10 个交易日，避开短期反转危险区，提取最纯粹的中期趋势。</p>

<p><strong>优化方案 2：</strong><code>ts_sum(returns &gt; 0 ? 1 : 0, 250)</code><br>
"Quality of Momentum" 高质量动量——上涨天数多的股票趋势稳定，动量延续概率大。</p>
{% endhideToggle %}
<!-- endtab -->

<!-- tab 🔄 反转 Reversal -->
<h3>Reversal Strategy</h3>
<p>理解为<strong>"均值回归"</strong>——价格与当前主导 trend 负向变动，也叫回调 Correction 或反弹 Rally。</p>

<h2>四、数据处理：Cross-sectional vs Time-series</h2>

{% tabs CS截面 vs TS时序 %}
<!-- tab Cross-sectional 横截面 -->
<h3>Cross-sectional 截面操作</h3>
<p>在任意一个时间点，对股票池 Universe 中<strong>所有股票同时执行计算</strong>。分析某一公司同一时刻与 peers 的对比。</p>
<p>常用算子：<code>rank</code>、<code>z-score</code>、<code>scale</code>、<code>sign</code></p>
<!-- endtab -->

<!-- tab Time-series 时序 -->
<h3>Time-series 时序操作</h3>
<p>对<strong>单只股票的历史时间窗口</strong>内执行计算。分析特定变量如何随时间变化。</p>
<p>常用算子：<code>ts_delta</code>、<code>ts_delay</code>、<code>ts_sum</code>（<code>ts_</code> 前缀）</p>
<!-- endtab -->
{% endtabs %}

<p>💡 <strong>截面和时序可以嵌套组合</strong>，称为 Nested Factor Construction。</p>

<h2>五、Neutralization 中和（风险隔离）</h2>

<p>绝大多数股票随大盘同涨同跌，我们需要挖掘<strong>不受大盘影响的 Alpha</strong>。采用 Long-Short 多空对冲策略消除市场 β：做多因子得分高的股票，做空得分低的股票，且做多和做空资金总额相等（<strong>Dollar Neutral</strong>）。</p>

{% hideToggle ⚙️ Brain 平台 Market Neutralization 三步流程（点击展开） %}
<ol>
  <li><strong>均值中心化</strong>：所有股票信号之和为 0，实现 Dollar Neutral</li>
  <li><strong>L1-Normalization 归一化</strong>：将分数转化为权重，权重和为 1</li>
  <li><strong>Allocation 分配</strong>：根据归一化后的 weight 分配实际金额</li>
</ol>
{% endhideToggle %}

<h2>六、Decay 衰减 & Truncation 截断</h2>

{% tabs Decay vs Truncation %}
<!-- tab Decay 衰减 -->
<h3>Decay 衰减</h3>
<p>控制<strong>换手率 Turnover</strong> 的核心算子。本质是线性加权平滑：</p>
<ul>
  <li>✅ 有效降低换手率，保证组合稳定性</li>
  <li>⚠️ 存在 Phase Lag 相位延迟——市场反转剧烈时信号反应慢</li>
  <li>📏 一般 decay 天数设为 <strong>2 ~ 10</strong></li>
</ul>
<!-- endtab -->

<!-- tab Truncation 截断 -->
<h3>Truncation 截断</h3>
<p>限制单个股票在 Portfolio 中的<strong>权重上限</strong>，防止特质性风险 Idiosyncratic Risk。避免某一只主力股意外导致整个组合崩溃。一般设为 <strong>0.05 ~ 0.1</strong>。</p>
<!-- endtab -->
{% endtabs %}

<h2>七、数据清洗三机制</h2>

<ol>
  <li><strong>Pasteurization 巴氏化</strong>：Off 时代码可能用到股票池以外的数据；On 则严格限制</li>
  <li><strong>Unit Handling 单位处理</strong>：不同单位（美元 vs 百分比）错误相加时系统警告</li>
  <li><strong>NaN Handling 缺失值处理</strong>：保持 NaN 或用 0 填充</li>
</ol>

{% hideToggle 📌 Test Period 样本外测试（点击展开） %}
<p>机器学习和量化中极其重要的<strong>样本外测试 (Out-of-Sample, OOS)</strong> 机制，对应 ML 中测试集/验证集概念，避免 Overfitting。</p>
{% endhideToggle %}

<h2>九、Operators 算子七大类</h2>

{% tabs 七大算子类型 %}
<!-- tab 算数 & 逻辑 -->
<h3>1. 算数算子</h3>
<p>加减乘除、<code>abs()</code>、<code>sign()</code> 等。构建基础财务比率或价差。</p>

<h3>2. 逻辑算子</h3>
<p>条件判断返回 1(True) 或 0(False)。<code>&lt;</code> <code>&gt;</code> <code>==</code> <code>if_else</code>。构建 mask 或过滤器。</p>
<!-- endtab -->

<!-- tab 时序 & 截面 -->
<h3>3. 时序算子</h3>
<p>同一股票过去 d 天纵向计算，前缀 <code>ts_</code>。如 <code>ts_delta</code>、<code>ts_delay</code>、<code>ts_sum</code>。</p>

<h3>4. 截面算子</h3>
<p>同一时间点全市场股票处理。如 <code>rank()</code>、<code>zscore()</code>。</p>
<!-- endtab -->

<!-- tab 向量 & 转换 -->
<h3>5. 向量算子</h3>
<p>多维数据降维压缩。如 <code>vec_avg()</code>、<code>vec_sum()</code>、<code>vec_max()</code>。</p>

<h3>6. 转换算子</h3>
<p>改变数据分布：<code>log(x)</code>、截断极端值、非线性→线性化。基本面数据有长尾分布，投入模型前先正态化。</p>
<!-- endtab -->

<!-- tab 分组 -->
<h3>7. 分组算子</h3>
<p>结合截面逻辑，计算范围限制在特定类别（行业、板块）。前缀 <code>group_</code>。</p>
<!-- endtab -->
{% endtabs %}

<h2>十、实战专题：VWAP 与 CLV</h2>

{% tabs VWAP vs CLV %}
<!-- tab VWAP -->
<h3>VWAP 成交量加权平均价</h3>
<p><strong>公式：</strong><code>sum(price × volume) / sum(volume)</code>，代表全天真实资金共识成本。</p>

<p><strong>VWAP/close 的两个特点：</strong></p>
<ol>
  <li><strong>Reversion</strong>：比较"全天平均成本"和"最终收盘价"。若 vwap/close &gt; 1（收盘价低于 VWAP），尾盘砸盘，价格跌破全天公允价值。基于反转理论，明天大概率涨回来 → 做多信号。</li>
  <li><strong>High Turnover</strong>：日内级别短期因子，每天尾盘情绪都在变。</li>
</ol>
<!-- endtab -->

<!-- tab CLV -->
<h3>CLV 收盘位置值</h3>
<p><strong>公式：</strong><code>CLV = ((Close - Low) - (High - Close)) / (High - Low)</code></p>

<ul>
  <li><strong>(Close - Low)</strong>：多头战果</li>
  <li><strong>(High - Close)</strong>：空头战果</li>
  <li><strong>分子 = 多头 − 空头</strong>：正 = 多头胜，负 = 空头胜</li>
  <li><strong>分母 = High - Low</strong>：全天总振幅，用于归一化</li>
</ul>

<p>CLV 取值 <strong>[-1, 1]</strong>：</p>
<ul>
  <li><strong>CLV = 1</strong>：光头阳线，极度亢奋</li>
  <li><strong>CLV = -1</strong>：光脚阴线，极度悲观</li>
  <li><strong>CLV = 0</strong>：两方打平，市场犹豫震荡</li>
</ul>

<p>💡 <strong>结合 Reversion 理论</strong>：CLV ≈ -1 时，恐慌盘可能已出清，明天大概率超跌反弹。</p>

<h2>十一、技术分析三大经典指标</h2>

{% hideToggle 📊 RSI / Bollinger / MACD 详解（点击展开） %}
<h3>1. RSI 相对强弱指数</h3>
<p>上涨/下跌幅度比例，市场<strong>"情绪温度计"</strong>。</p>

<h3>2. Bollinger Bands 布林带</h3>
<p>上下加减两倍标准差作为边界，预测均值回归和反弹。</p>

<h3>3. MACD 平滑异同移动平均线</h3>
<p>趋势测速——比较短期均线和长期均线的距离来判断趋势方向。</p>

<blockquote>
  <p>"It's based on the assumption that market participants' psychology and behavior patterns repeat..."</p>
  <p>技术分析核心哲学：所有利好、利空、恐慌、贪婪，最终都变成实打实的 Bid 和 Ask，体现在 Price 和 Volume 上。人类追涨杀跌的本性从未改变，过去的量价形态在未来大概率重演。</p>
</blockquote>
{% endhideToggle %}

<h2>十二、Trade_when 持仓算子</h2>

<p><strong>语法：</strong><code>trade_when(entry_condition, Alpha, exit_condition)</code></p>

<table>
<tr><th>条件</th><th>行为</th></tr>
<tr><td>entry = True</td><td>更新仓位，输出最新 Alpha 值</td></tr>
<tr><td>exit = True</td><td>平仓清空，输出 NaN</td></tr>
<tr><td>两者皆 False</td><td>保持持仓，输出上次进场的 Alpha</td></tr>
</table>

<p>⚠️ 算式中条件越多，越容易 Overfitting。</p>

{% hideToggle 💡 实战案例（点击展开） %}
<p><code>positive_days = ts_sum(returns&gt;0 ? 1 : 0, 250);</code><br>
<code>trade_when(volume&gt;adv20, positive_days, -1)</code></p>
<p>Alpha 信号为上涨天数。进场条件："交易量突破 20 日均线"。离场信号设为常数 -1——永不主动平仓，避免 Overfitting 同时降低 Turnover。</p>
{% endhideToggle %}
